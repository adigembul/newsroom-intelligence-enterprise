1

Mulai dari query yang spesifik

Gunakan nama lembaga, tokoh, lokasi, dan isu. Contoh: banjir Jakarta Selatan, kurikulum digital, atau harga beras.

2

Pilih sumber sesuai kebutuhan

GDELT/RSS untuk berita, Bluesky untuk percakapan sosial publik, Hacker News untuk isu teknologi, Apify/Social X untuk data hasil scraper. GDELT memakai cache otomatis; jangan refresh manual berulang-ulang dalam beberapa detik.

3

Baca alert negatif sebagai sinyal awal

Sentimen negatif bukan otomatis buruk atau hoaks. Gunakan sebagai prioritas follow-up, lalu verifikasi ke narasumber primer.

4

Gunakan report untuk rapat redaksi

Buka tab Analytics Report, cek top viral, domain dominan, rekomendasi redaksi, lalu download HTML report seperti dashboard Hootsuite.

5

Cek hoaks sebelum publikasi

Tempel klaim viral di tab Cek Hoaks. Sistem memberi bukti terbuka, sinyal risiko, dan checklist verifikasi.

6

Konversi hasil scraper

Upload CSV/JSON dari Apify atau Social X. Aplikasi menormalkan kolom seperti text, url, author, createdAt, likes, comments, shares, dan views.

SOP cepat untuk jurnalis

  1. Scan isu selama 60 detik atau lebih untuk memetakan narasi awal dan menghindari rate limit API publik.
  2. Ambil 3 konten viral teratas sebagai bahan angle dan kontra-angle.
  3. Gunakan keyword dominan untuk mencari dokumen, data primer, dan narasumber.
  4. Jalankan cek hoaks pada klaim paling sensitif.
  5. Tulis rilis/berita dengan 5W+1H, lalu lampirkan ringkasan analitik ke editor.
